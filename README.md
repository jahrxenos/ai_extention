# Адаптация текста

Браузерное расширение для Firefox, которое упрощает сложный текст на любой веб-странице с помощью AI. Полезно при чтении научных статей, документации, нормативных документов.

## Возможности

- Три уровня упрощения: лёгкое, среднее, максимальное
- Адаптация выделенного фрагмента или всей страницы
- Откат к оригиналу — глобальный или для каждого фрагмента отдельно
- Использует Groq API (бесплатный тариф)
- Не требует сервера — запросы идут напрямую из браузера

## Пользователям

### Установка

1. Скачайте и распакуйте архив с расширением
2. Откройте в Firefox: `about:debugging` → **Этот Firefox** → **Загрузить временное дополнение**
3. Выберите файл `manifest.json` из папки расширения

### Настройка API-ключа

1. Зарегистрируйтесь на [console.groq.com](https://console.groq.com/keys) и создайте бесплатный ключ
2. В браузере нажмите на иконку расширения → **⚙ Настройки**
3. Вставьте ключ в поле **API-ключ** и нажмите **Сохранить**
4. Кнопка **Проверить ключ** убедится, что ключ работает

### Использование

1. Откройте любую веб-страницу
2. Правый клик → **Адаптировать текст** (или нажмите иконку расширения на панели)
3. В появившейся панели выберите уровень упрощения:
   - **Лёгкая** — убрать термины, сохранить все детали
   - **Средняя** — простой язык, ключевая информация
   - **Максимальная** — объяснить как пятикласснику
4. Нажмите **Выделенное** (для выделенного фрагмента) или **Вся страница**
5. Для возврата к оригиналу нажмите **↩ Оригинал** в панели или кнопку рядом с каждым фрагментом
6. Закрыть панель (✕) — автоматически откатывает все адаптации

## Разработчикам

### Требования

- Firefox 91+
- Groq API ключ

### Структура проекта

```
├── manifest.json          # Конфигурация расширения (MV2, Firefox)
├── background/
│   └── background.js      # Фоновый скрипт: меню, API-запросы, парсинг ответа
├── content/
│   ├── content.js         # Контент-скрипт: панель, адаптация текста в DOM
│   └── content.css        # Стили панели и адаптированных фрагментов
├── options/
│   ├── options.html       # Страница настроек
│   ├── options.css        # Стили настроек
│   └── options.js         # Логика настроек: сохранение ключа, тест API
└── icons/                 # Иконки расширения
```

### Архитектура

**background.js** — единственный процесс с доступом к сети:
- регистрирует пункт контекстного меню и обработчик кнопки панели
- при активации инжектирует контент-скрипт и CSS в активную вкладку
- принимает сообщение `groq` от контент-скрипта, отправляет запрос к Groq API, парсит нумерованный ответ и возвращает массив адаптированных текстов
- принимает сообщение `openOptions` и открывает страницу настроек

**content.js** — выполняется в контексте страницы:
- создаёт плавающую панель управления (один раз, при первом вызове)
- в режиме **выделения**: берёт текущий Selection, отправляет его в background, заменяет содержимое Range на `<mark>` с адаптированным текстом
- в режиме **страницы**: собирает до 30 видимых блоков `p, li, h2–h4, blockquote, td` длиннее 80 символов, отправляет пакетом, заменяет содержимое
- хранит массив `replacements` с функциями отката для каждого блока

**Поток данных:**

1. Пользователь нажимает кнопку в панели
2. `content.js` собирает текст (выделение или блоки страницы) и отправляет сообщение в `background.js`
3. `background.js` формирует запрос к Groq API и получает адаптированные тексты
4. `background.js` возвращает результат обратно в `content.js`
5. `content.js` заменяет текст на странице и сохраняет оригинал для возможного отката

### Локальная разработка

```bash
git clone https://github.com/jahrxenos/ai_extention.git
cd ai_extention
```

Загрузите расширение через `about:debugging` → **Загрузить временное дополнение** → `manifest.json`.

После изменения файлов нажмите **Перезагрузить** в `about:debugging`. Контент-скрипты обновятся при следующем открытии панели на странице.

### API

Расширение использует [Groq API](https://console.groq.com), совместимый с OpenAI Chat Completions:

- Модель: `llama-3.1-8b-instant`
- Температура: `0.3`
- Формат запроса: нумерованные фрагменты `[1] текст\n[2] текст`
- Формат ответа: те же нумерованные фрагменты на языке оригинала
